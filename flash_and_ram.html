Flash and RAM in Embedded Systems – A Guide for Freshers

🧠 What is Memory in Embedded Systems?

Microcontrollers use internal memory for two key purposes:

  • Flash: To store the program code and constants.
  • RAM: To store variables and runtime data.

🔹 Flash Memory – The Non-Volatile Storage

FeatureFlash Memory
Non-volatile?✅ Yes (retains data after power off)
UsageStores program code (firmware), constants
Write/EraseSlower, sector-based
Write LifetimeLimited (~10k-100k cycles)

🧩 Analogy: Like a pen drive—it holds data even after shutdown.

📦 Example Code (Flash)

const char firmware_version[] = "v1.0.5";

🔸 RAM – The Volatile Working Area

FeatureRAM
Volatile?✅ Yes (data lost on power off)
UsageStack, heap, buffers, local/global variables
Speed✅ Fast

🧩 Analogy: Like your work desk—everything is cleared when power is lost.

📦 Example Code (RAM)

char rx_buffer[64];  // Stored in RAM for UART communication

Common Sections:

SectionStored InDescription
.textFlashCode (functions)
.rodataFlashConstants
.dataRAMInitialized variables
.bssRAMUninitialized variables

⚙️ Boot Process After Power-On

  1. MCU starts from 0x00000000 (usually Flash)
  2. Reads vector table
  3. Executes Reset Handler
  4. Copies data to RAM and calls main()

✅ Flash vs RAM Summary

FeatureFlashRAM
VolatileNoYes
StoresCode, constantsVariables, stack
SpeedSlowerFaster
SizeLarger (e.g., 512KB)Smaller (e.g., 64KB)

🛠 Tips for Embedded Freshers

  • Use const to move data to Flash.
  • Avoid deep function calls to prevent stack overflow.
  • Check memory usage with .map files.
  • Understand linker script memory assignments.
